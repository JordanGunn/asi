# Implementation Documentation Index

This section defines the implementation-facing layer of ASI: how maintainers enforce existing design and specification contracts in real workflows.

It complements, but does not replace, the normative sources in `docs/design/specs/`.

Audience: maintainers and reviewers implementing or evolving ASI skills.

## Normative Language

The following terms are normative when used in requirement statements:

- `MUST`: mandatory requirement for compliance.
- `SHOULD`: recommended default; deviations require explicit justification.
- `MAY`: optional behavior that does not affect baseline compliance.
