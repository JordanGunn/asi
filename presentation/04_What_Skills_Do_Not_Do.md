# What Skills Do *Not* Do

## Skills are not magic

- Do **not** run automatically because they feel relevant
- Do **not** make models “smarter”
- Do **not** guarantee correct outcomes for vague input

## Skills are not just prompts

- A prompt describes intent.
- A skill should define **contracts**, **scope boundaries**, **failure semantics**, and often **deterministic entrypoints**.

## Skills should not wrap pure determinism

If there’s exactly one correct outcome given the input:

- Prefer a **script/tool** (not a skill)
